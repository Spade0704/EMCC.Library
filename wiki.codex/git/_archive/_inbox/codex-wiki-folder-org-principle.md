# Codex Wiki — Folder-Organization Principle (Design Record)

**Status:** Design locked 2026-05-22 (first dogfood: codex-self / `EMCC.Library/wiki`).
**Scope:** GENERAL principle for all Codex wikis — not specific to EMCC.Library. EMCC.Library is the worked instance.
**Implementation:** Phase 1 (logical) NOW; Phase 2 (physical regroup) DEFERRED as a future `PROJECT_WIKI_BUILD_SPEC §2.2` amendment.
**Origin:** Surfaced by the first dogfood. The flat `_`-prefixed §2.2 layout doesn't cleanly express the desired separation of reader content / backend / data, nor the git-tracked vs local-confidential split.

---

## The principle

Every Codex wiki separates concerns along two orthogonal axes plus confidentiality:

- **Audience** — reader-facing wiki content vs backend/system files the wiki user never touches.
- **Git** — tracked (in repo) vs ignored (local-only).
- **Confidentiality** — shareable vs confidential (usually aligns with the git axis).

Three logical buckets:
1. **Wiki content** — the reader-facing reading layer. Tracked.
2. **System** — backend: automation scripts, generated dashboards, agent config/context, persona.
3. **Wiki data** — canon, decisions, sources, inbox, confidential, brain-dump.

---

## Locked decisions (2026-05-22)

1. **STANDALONE wiki model.** The wiki repo must reproduce its own behavior from a fresh clone. → Load-bearing + input + code files are TRACKED, including `_config/`, `_context/` (incl. the verbatim-ship `INGEST_PROCEDURE.md` / `SEMANTIC_LINT_PROCEDURE.md`), `.claude/personas/`, AND `_scripts/` (the wiki carries its own runnable copy). Only *pure outputs*, *ephemeral*, and *confidential* are ignored.
2. **`_sources/raw/` = single tree** (no confidential/git split). Preserves the `canon_sources` single-path citation contract (cite `_sources/raw/...`, never forked).
3. **`EMCC.Library/` is a git repo now** — repo root; `.gitignore` lives here. Tracks main-project docs (e.g. `0. Inbox/`) plus a curated subset of `wiki/`. **Supersedes** the path-redirect amendment's "`.gitignore` excludes `wiki/` entirely" — the wiki is now a partially-tracked member of the EMCC.Library repo, not a wholly-ignored artifact.

---

## Two-phase implementation

### Phase 1 — logical (NOW)
Keep the existing flat `_`-folder §2.2 layout untouched. Express the principle via:
- `.gitignore` at the repo root (`EMCC.Library/.gitignore`).
- Obsidian "excluded files" (Settings → Files & Links) to hide backend/data folders from the reading view.

Zero script changes, zero spec-structure change. Validates the separation on the live dogfood.

### Phase 2 — physical (FUTURE; deferred spec amendment)
Regroup into:

```
<project>/wiki/
├── content/          reader-facing ("the wiki"; Obsidian vault root)
├── System/           backend (regenerable + load-bearing)
└── wiki_data/
    ├── git/          tracked data
    └── local/        gitignored data (confidential / ephemeral)
```

**Cost:** `PROJECT_WIKI_BUILD_SPEC §2.2` rewrite + path-math across bootstrap.py + ~all 15 scripts (they hardcode `_canon/ _config/ _dashboards/ _sources/raw/` relative to `WIKI_ROOT`) + `canon_sources` scheme + every template + re-bootstrap. Own task arc; wiki-spec version bump.
**Trigger:** only if the flat layout proves annoying in practice. Otherwise Phase 1 stands.

## Phase-1 git disposition (standalone, one tree)

Repo root = `EMCC.Library/`; paths repo-root-relative.

**TRACK**
- `0. Inbox/` + other main-project docs (project level — distinct from wiki `_inbox/`)
- `wiki/Home.md`, `wiki/README.md`
- `wiki/00-Start-Here/`, `wiki/01-Architecture/`, `wiki/02-Operations/`, `wiki/04-Contributing/`
- `wiki/Attachments/`, `wiki/public/`, `wiki/scripts/` (consumer's own exporters)
- `wiki/_canon/`, `wiki/_decisions/`
- `wiki/_config/`, `wiki/_context/`, `wiki/.claude/personas/` ← tracked because standalone
- `wiki/_scripts/` (+ `_lib/`) ← tracked because standalone (self-contained clone)
- `wiki/_sources/raw/` ← single tree, tracked

**IGNORE**
- `wiki/_dashboards/` — generated by `update_dashboards.py`
- `wiki/_inbox/` — ephemeral triage
- `wiki/_confidential/` — secrets
- `wiki/_brain_dump/` — quarantined ideas *(JUDGMENT CALL — confirm track vs ignore)*

**Proposed `EMCC.Library/.gitignore`:**

```gitignore
# --- Codex wiki: generated / ephemeral / confidential (not for git) ---
wiki/_dashboards/
wiki/_inbox/
wiki/_confidential/
wiki/_brain_dump/

# --- Obsidian noise ---
wiki/.obsidian/workspace*
.DS_Store
```

Everything else is tracked by default. `_scripts/` + `_dashboards/` regenerate via Codex sync / `update_dashboards.py`, but under the standalone model `_scripts/` is kept in-repo so a clone is runnable without the module present.
