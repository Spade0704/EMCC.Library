# Obsidian Setup for Project Codex

## How to open this as Obsidian Vault

1. Open Obsidian
2. Click **Open folder as vault**
3. Select the root of this repository (`EMCC.Library` folder)

**Recommended:**
- Use the whole repo as vault, or
- Use `wiki.codex/git/` as a focused vault for the Codex dogfood content

## Recommended Plugins
- Docxer (for .docx preview)
- HTML Reader
- Advanced Tables
- Dataview (very useful for Codex)

## Folder Structure (post-S002 v1.1 canonical layout)
- `wiki.codex/git/raw/` → Original source files (DOCX / PDF / spec exports)
- `wiki.codex/git/codex/` → Codex authoritative spec docs (`CODEX_BUILD_SPEC_v1_3.md`, `CODEX_LIBRARIAN.md`, procedures, this guide)
- `wiki.codex/git/00-Start-Here/` / `01-Architecture/` / `02-Operations/` / `04-Contributing/` → Dogfood wiki topic pages
- `wiki.codex/local/` → Private / gitignored content (brain-dump, unfiled notes)
- `0-Inbox/` → Quick capture awaiting Librarian sort
- `Biz.Automation/wikisys.library/` → Codex automation engine (scripts, templates, config, canon)

> Lattice 3.0 protocol docs live in `EMCC.DFDU/documents/lattice/` (DFDU repo), not Library. Open DFDU as a separate vault when reviewing Lattice protocol material.
