THE FIVE BUILD PRINCIPLES (every task)
--------------------------------------------------
1. Think Before Coding   — state assumptions; ask if unclear
2. Simplicity First      — pure stdlib only; no speculative abstractions
3. Surgical Changes      — "Out of scope" list mandatory; don't drift
4. Goal-Driven Execution — success criteria must be verify-mechanical
5. Spec Wins             — spec contradicts plan? Stop. Ask. Don't decide.

Tradeoff: trivial edits (typo, comment fix) skip the full format.
Non-trivial work — always full rigor.

VERIFYING (without shipping)
--------------------------------------------------
  /codex-plugin:verify

Runs 5-stage check on Codex:
  1. Stdlib-purity: scan _scripts/ for non-stdlib imports
  2. Self-test bootstrap: python bootstrap.py to a temp path
  3. Dashboards: python _scripts/update_dashboards.py on the test wiki
  4. Validator pass: confirm 0 errors, 0 warnings
  5. Idempotency: re-run bootstrap; confirm identical output

Scope-aware: skips stages 2–5 for docs-only / template-only diffs.
No commit. Use when you want to know "would this ship?" before deciding.

Manual fallback (if no plugin yet):
  - Stdlib check: grep -rn "^import\|^from" _codex/_scripts/ and eyeball
  - Self-test: python bootstrap.py %TEMP%/codex-selftest && \
               cd %TEMP%/codex-selftest && python _scripts/update_dashboards.py

ENDING A SESSION
--------------------------------------------------
Just one command:

  /codex-plugin:go

Runs 8 steps automatically:
  1. Verify (re-runs /codex-plugin:verify)
  2. Simplify + Surgical Changes check (diff traces to plan?)
  3. Session log entry → tasks/sessions.md (which P# items shipped)
  4. Lessons check (append if pattern emerged)
  5. R_STATE sync → CLAUDE.md (advance current P# cursor)
  6. Commit (Conventional Commits format)
  7. Push to main
  8. Report SHA + next P# + manifest progress

STOP-and-ask triggers:
  - Verify fails (any of the 5 stages)
  - Stdlib purity violated (external import detected)
  - Spec contradicts the change
  - Idempotency broken (bootstrap not reproducible)
  - Scope exceeded / diff > 2x expected size
  - Lesson violated
  - Score-vs-priority mismatch in build plan still unresolved


AUDITS
--------------------------------------------------
  Run [SPEC|STDLIB|IDEMPOTENCY|CROSS-PLATFORM|DX] audit on [scope].
  Fix issues automatically. Score out of 10.
  Append to tasks/sessions.md with [AUDIT] tag.
  Then /codex-plugin:go.

Audit types

  SPEC           — does the build match spec §X? Frontmatter complete?
                   Hard rules from §8 honored?
  STDLIB         — any non-stdlib imports? PyYAML sneak in anywhere?
  IDEMPOTENCY    — bootstrap twice = byte-identical output?
                   Sync twice = no spurious changes?
  CROSS-PLATFORM — pathlib used everywhere? "|"/"/" handled correctly?
                   Windows + POSIX paths both work?
  DX             — --dry-run + --help on every entry point?
                   Error messages clear? Dashboards readable?
                   Operational caveats commented inline?

CODEX HARD RULES (spec §8 — never break)
--------------------------------------------------
1. Pure Python stdlib only. No pip install. No PyYAML.
2. Never edit files outside D:\Claude Projects\Project Codex\
   (exception: self-test bootstraps to a temp path).
3. Always run a self-test bootstrap before declaring complete.
4. Always show the file manifest at the end of the build.
5. INGEST_PROCEDURE.md and SEMANTIC_LINT_PROCEDURE.md ship VERBATIM —
   never paraphrase, shorten, or "improve" them.
6. The spec wins every contradiction. Flag and ask, don't decide.
7. No Iron Soul reference is provided in this project — build from
   spec alone. Iron Soul is the first consumer, not a derivation source.


RED FLAGS (stop and ask)
--------------------------------------------------
- A request not in the spec — confirm scope before adding
- A script seemingly needing a non-stdlib import — find a stdlib path or escalate
- The spec is genuinely ambiguous on a buildable detail — surface it,
  propose a resolution, wait for confirmation before writing
- Score-vs-priority mismatch in build plan — pause, resolve design first
- Required input missing (currently none — both procedure files are present)
- A v1.1-novel artifact has no Iron Soul precedent (check_concept_coverage.py,
  scaffold_source.py, the two _context/ procedures) — implement from spec alone
