## Framed decision

`check_cross_refs._resolve_target(target, stem_to_page, relpath_to_page)` resolves `[[wikilinks]]`:
bare `[[Page]]`→by filename stem; path-qualified `[[Folder/Page]]`→by full wiki-root-relative path; leading `./` stripped (root-relative); **no `../` handling** → `[[../folder/Page]]` always reported broken. Consumer residehub wrote 28 valid links as `[[../folder/Page]]` pointing at sibling folders relative to the linking page; all 28 false-flagged.

**Option A (page-relative):** thread linking page's dir in; `..`-containing targets resolve relative to the page's folder, collapse `.`/`..`, return broken if escapes wiki root. Bare/`Folder/Page`/`./` unchanged.
**Option B (redundant-to-root):** strip leading `../`, resolve root-relative.

## Stage 2 — independent advisor analyses (Opus 4.8)

### The Breaker
Sketch mostly holds; found edge traps. (1) `.md` strip-first is safe — keep order. (2) Empty-key collapse (`[[../..]]` → `""`) must be guarded before lookup, don't trust luck-of-naming. (3) `..` mid-path (`foo/../bar`) and literal `my..weird` folder are safe **only if** the impl uses `seg == ".."` segment-equality, never `".." in seg` substring. (4) Root page `../x` → escapes → None → correct. (5) Trailing `..` resolves to a folder key → None (flag dir-vs-page collision, low risk). (6) **Windows backslash hole** — `[[..\folder\Page]]` splits as one segment, won't resolve; add a test asserting backslash targets are NOT silently resolved (consistent with today, not a regression). Must-add tests: `[[../..]]`→None, over-pop→None, root-page `../x`→None, `foo/../bar`→bar, `my..weird` literal, empty-key guard.

### The Simplifier
The hand-rolled push/pop loop **is** `posixpath.normpath` minus the escape check. `posixpath` is `os.path`'s POSIX sibling, available on Windows, pure-string, uses `/`, collapses `..`:
```python
joined = posixpath.normpath(posixpath.join(base_dir, target_path))
if joined == ".." or joined.startswith("../"):
    return None  # escaped root
```
That's the whole normalizer — delete `_normalize_relparts` (unearned abstraction reimplementing stdlib). `normpath` leaves leading `..` in place = exactly the escape signal. Threading `page_dir` is the minimal seam (data isn't otherwise present).

### The Surgeon
Directive is one verb: *normalize `../`*. (1) **`./` is OUT** — touching it is scope-creep; its strip→root-relative is the established contract. (2) page_dir threading is additive/low-risk — **grep for other callers first** (orphan-clearing consumes the return, not the signature). (3) Docstring correction of the `../` line is doc-sync, not creep. (4) **Silent finding-flip is the real hazard** — a consumer may have `[[../x]]` links currently-broken that authors expect broken; flipping them Syncs to 4 wikis → needs Auditor + consumer-impact note. (5) Spec: if CODEX/PROJECT spec documents resolvable link forms, update in the same atomic commit; if not, surface as finding. Minimal line: code (`..` only) + docstring + spec-if-exists + disclosure.

### The Reviewer
(1) The hybrid (`folder/Page` root-relative but `../folder/Page` page-relative) is a **third dialect** — in every path language `../foo` is relative to the same base as `foo`; authors will hit a correctness trap. (2) `./` resolving root-relative is **actively misleading** (`./` literally means "here"). (3) Name `_normalize_relparts` is dishonest — it *resolves against a base*; call it `_resolve_page_relative(linking_page_dir, target)`. (4) Dashboard `target not found` is useless for an escaped `../` — wants distinct "escapes wiki root" message. (5) Honest fix: make `./` and `..` BOTH page-relative.

### The Pragmatist
(1) A & B **coincide only when the page is exactly one level deep**; diverge at 2+ deep (`a/c/X.md` + `../Page`: A→`a/Page` correct; B strips→`Page` wrong/false-broken) and on `../../` (B silently resolves a genuine escape). **B is a coincidence that rots silently as the tree deepens** — a Level-2 bug Synced everywhere. (2) Smallest *correct* diff = A, ~6-8 lines, `os.path.normpath` of the join; `./`-strip precedent already exists. (3) A is purely additive — bare/`Folder/Page`/`./` hit unchanged branches → 761 tests can't regress; B touches the shared branch. **Ship A.**

## Stage 3/4 — peer review + chairman synthesis

**Unanimous agreement:** Option A. The Pragmatist's depth-divergence proof + the Breaker's escape analysis make B indefensible — B silently mis-resolves at tree depth ≥2 and resolves genuine root-escapes, then Syncs that bug to every consumer.

**Clash — should `./` also become page-relative? (Reviewer/Simplifier YES, Surgeon NO).**
Resolved **NO (Surgeon wins) for this commit.** The Reviewer's coherence argument is real but cuts against an *established Codex contract*: path-qualified `Folder/Page` is root-relative by deliberate design (spec §"Wikilink format" + the disambiguation generator emits root-relative paths; existing test `test_path_qualified_wikilink_resolves` links `[[01-Domain/Foo]]` from `01-Domain/Linker.md` and expects root-relative — page-relative would break it). Within that contract, `..` MUST be page-relative (no other way to express upward traversal), so the hybrid is *inherent to Codex's root-default*, not a new invention. Flipping `./` from root-relative to page-relative is exactly the Surgeon's #1 hazard (silent finding-flip across 4 consumer wikis) and is out of the directive's stated scope. **→ `./` frozen; logged as a separate backlog finding.**

**Adopted from advisors:**
- **Simplifier:** use `posixpath.normpath(posixpath.join(page_dir, target))` — no hand-rolled normalizer. Escape = result `== ".."` or `startswith("../")` → None.
- **Breaker:** trigger on segment-equality `".." in t.split("/")` (covers leading + mid `..`, immune to `my..weird`); guard empty/`.` key; tests for over-pop, root-page escape, `foo/../bar`, `.md` ext, literal `..`-substring folder, backslash-not-resolved.
- **Reviewer:** rename helper `_resolve_page_relative`; add one docstring sentence stating the semantic explicitly.
- **Surgeon:** single caller confirmed (grep: only line 144); docstring update in-scope; **spec carries no resolution rules** (only generation/format §"Wikilink format") → no spec change, surfaced as finding.

**Deferred (gold-plating for this atomic commit):** Reviewer's distinct "escapes wiki root" dashboard message — expands the Pattern-#5 locked dashboard literal + its tests. Escape → `None` → existing "target not found" path. Logged as optional follow-up.

**Recommendation:** Option A, `posixpath`-based, `..`-segment-triggered, scope frozen to `..`. **First test to write:** sibling resolve — page at `01-Domain/Linker.md` with `[[../02-Other/Baz]]` resolves to `02-Other/Baz.md` (and the depth-2 + escape-root cases the Pragmatist/Breaker flagged).
